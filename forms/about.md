# About

## What is this?

A hotel booking app described in 6 markdown files, rendered by 285 lines of JavaScript. No framework. No server. No database. No build step. Deployed on GitHub Pages for free.

## The alternative

"Vibe code a hotel booking" generates thousands of lines — React components, Express server, database schema, auth middleware, Stripe integration, session management, error handling. 10-20 files, 3000+ lines, multiple dependencies, a build step, a server to run. Probably 200-500k tokens across multiple sessions. Half of that debugging.

## This approach

6 markdown files. 285 lines of JS. One HTML file. ~5-10k tokens to build. No bugs to debug because there's barely any code to be buggy.

The markdown files are readable by anyone. The vibe-coded version is readable by the AI that wrote it.

## Less code, fewer tokens, fewer bugs

The economics compound. Less code means fewer tokens to generate it, fewer bugs to debug, fewer tokens to fix them, less infrastructure to maintain, less cost to run.

The t-shirt says "285 lines."
